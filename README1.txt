В ходе выполнения лабораторной работы №2 я исследовала эффективность параллельного умножения матриц с использованием OpenMP при различном количестве потоков (1, 2, 4, 8) и размерах матриц (200, 400, 800, 1200, 1600, 2000).
Результаты экспериментов:
1. Размер матрицы N=800 показал наилучшее ускорение:
- При 8 потоках ускорение составило 2.15x
- При 2 потоках ускорение составило 1.62x
- При этом размере данные эффективно используются процессором
2. При малых размерах матриц (N=200-400) ускорение минимальное:
- Ускорение не превышает 1.0-1.14x
- Время выполнения параллельной версии сопоставимо с последовательной
- Накладные расходы на работу с потоками влияют на общее время выполнения
3. При больших размерах матриц (N≥1200) ускорение также незначительное:
- Ускорение составляет 1.00-1.03x
- Объём данных превышает размер кэш-памяти процессора
- При N=1200 объём данных ~34.5 МБ, при N=2000 ~91.5 МБ
- Производительность ограничена пропускной способностью оперативной памяти
4. Производительность системы** во всех тестах составила 0.48-0.75 GFLOPS.

Параллелизация с помощью OpenMP эффективна при соблюдении двух условий:
1. Объём вычислений достаточен для компенсации затрат на создание потоков
2. Данные помещаются в кэш-память процессора

Для задач с большим объёмом данных простое увеличение количества потоков не даёт значительного ускорения. В таких случаях требуются дополнительные методы оптимизации.
